Command Line

The command line aka terminal is a text-based interface built into the computer's operating system. It is a way to interact with the operating system using pre-defined keywords called commands

MDN Docs
  • 'ls' lists the contents of the current file folder called a directory
  • 'cd' is used to change working (current) directory
  • 'cd ..' moves up one directory to the parent of the working directory
  • 'pwd' displays the current directory
  • 'mkdir' creates a new directory
  • 'touch' creates a file type. the file extension must be used when making the file

Git

Git is a way to store code in a repository, which could be private or public. Github is a popular website that allows programmer to store and share their code.

Github
  1. Click 'new' to create a new repository
  2. Name the repository. use dashes between words instead of spaces. This make it easier when you have to call the repository in the command line
  3. Click the 'README' box to include a README file
  4. Click 'create repository'
  5. To add the repository locally to your machine, hit 'code' button and under clone, click copy under HTTPS. A password protected method can be set up where the clone is copied as a SSH key
  6. Open a terminal on you machine and use 'cd' to change to the working directory where you want to store the repository locally
  7. use the command 'git clone' then paste the copied key from Github
  8. Now you can change to the directory and create directories and files uses commands
  9. To update changes made locally to git, use 'git add -A' to first add all changes
  10. Then use the command 'git commit -m "<message>"' This message should be a short description of the recent changes made to the program.
  11. The 'git push' will push the changes to the git repository (repo)
  12. To get updates from a repo not made locally, the command 'git pull orgin main' is used
